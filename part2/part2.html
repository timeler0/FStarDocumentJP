帰納的型によるデータ・証明・計算の表現

これまでに F* における 新しいデータ型の定義 を学びました。たとえば、次は要素の型 a をパラメータにもつリスト型です。

type list a =
  | Nil  : list a
  | Cons : hd:a -> tl:list a -> list a

これらの型に対してパターンマッチと再帰を使って基本的な関数を定義できることも見ました。例えば、次はリストの長さを計算する関数です。

let rec length #a (l:list a)
  : nat
  = match l with
    | [] -> 0
    | _ :: tl -> 1 + length tl

関数 length は list 型そのものとは別に、list の性質（長さ）を定義しています。しかし場合によっては、型の性質を型の定義と一緒に与えると便利です。例えば、リストの定義と同時に長さも持たせ、すべてのリストが構造的に長さの概念を備えるようにするのが自然なことがあります。次のようにします。

type vec (a:Type) : nat -> Type =
  | Nil : vec a 0
  | Cons : #n:nat -> hd:a -> tl:vec a n -> vec a (n + 1)

ここにあるのが最初のインデックス付き型 vec a n です。この定義は vec a : nat -> Type が型族を与える、と理解できます。すなわち vec a 0、vec a 1、… の各型で、どれも a 型の要素からなるリストを表しますが、インデックス n がリストの長さを表します。この vec の定義では、length 関数は冗長になります。v : vec a n があれば、再計算せずとも length v は n だと分かるからです。

型の性質を述べるためにインデックスで型定義を拡張するこのスタイルは、以前学んだ 内在的証明と外在的証明 を想起させます。すべてのリストに対して単一の list a を定義し、その長さを別（外在的）に length で計算する代わりに、vec ではリストの型そのものを内在的に拡張し、vec の型を見るだけで長さが分かるようにしています。

このような長さ付きの vec 型は、依存型プログラミングの入門でよく登場するので既に見たことがあるかもしれません。しかし、インデックス付き型の力はそれだけではありません。この節では、インデックス付き帰納的型を次の三つの観点から学びます。

  • データの表現：帰納的型により、リスト・ベクタ・木など様々なデータ型を構築できます。ケーススタディとして ベクタ と、暗号学的証明を備えた二分木構造である Merkle 木 を扱います。

  • 証明の表現：F* の核となる論理は、いくつかの単純な帰納的型定義の上に成り立っています。以前に見た論理結合子（命題論理の結合子 や 等式）を再訪し、それらが F* の原始概念ではなく、帰納的型に基づくいくつかの基本構成から定義されることを示します。言語の他の中核概念、例えば 停止性の扱い も、良基底再帰のモデル化 としての帰納的型の観点から理解できます。

  • 計算の表現：帰納的型定義により、他のプログラミング言語や計算モデルを F* に埋め込むことができます。ここでも二つのケーススタディを行います。

    • 単純型付きラムダ計算の深い埋め込み を、いくつかの簡約戦略とともに開発し、構文的型健全性を証明します。この例では、言語の構文を表す帰納的型、型体系を記述する関係、操作的意味論を記述する別の関係と、複数の帰納的型の活用を示します。

    • また、高階抽象構文 を用いて型付きラムダ項を表現する方法を示します。これは、関数をフィールドに持つ帰納的型の使い方を示す簡潔なスタイルです。

    • 最後に、構造化並行性をもつ命令的言語の浅い埋め込み を扱います。計算を無限分岐する帰納的定義木として表し、計算効果をモナドでモデル化する方法を導入し、高階での帰納的型の活用を示します。

このセクションは第 1 部よりやや高度です。F* の中核的な論理に関する技術的内容と、その概念が働く様子を示すケーススタディが交互に現れます。順に読み進めてもよいのですが、関心に応じて次の読み方のほうが取り組みやすいかもしれません。
