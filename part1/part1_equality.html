等値性（Equality）

等値性は、すべての依存型理論の設計に通底する繊細な論点であり、F* も例外ではありません。本章では、F* における 2 種類の等値性に手短に触れ、最も単純な用途に十分な基本情報を示します。後続の章 では、等値性をさらに深く扱います。

決定可能な等値性と eqtype

（factorial の定義などで）すでに暗黙に等値演算子 = を使ってきました。これは ブール の等値演算子です。2 つの項 e₁ : t と e₂ : t に対し、型 t が決定可能な等値性を備えていれば、(e₁ = e₂) : bool となります。

なぜすべての型が決定可能な等値性をもつわけではないかを見るため、int -> int のような関数型 t を考えてみましょう。2 つの関数 f₁, f₂ : int -> int が等しいかを判定するには、無限にあるすべての整数に適用して結果を比べる必要があります—これは明らかに決定不能です。

eqtype は、決定可能な等値性をサポートする“型の型”です。すなわち、e₁ : t と e₂ : t に対して e₁ = e₂ と比較できるのは、t : eqtype の場合に限られます。

任意の型定義について、F* はその型が eqtype かどうかを自動で判定します。詳細な判定方法は後で説明しますが、概略として、整数やブールなどの各種プリミティブ型が決定可能な等値性を備えることは F* にハードコードされています。新しい型を定義する際、F* はその型の値が、決定可能な等値性をもつ項から構造的に組み立てられているかを検査します。特に、e : t が関数を部分項として含みうる場合、その t は eqtype にはなりえません。

このため、決定可能な等値演算子の型は次のようになります。

val ( = ) (#a:eqtype) (x:a) (y:a) : bool

すなわち、x = y が型付け可能なのは、x : a かつ y : a、さらに a : eqtype の場合に限られます。

注釈

ここでは、F* の中置演算子を定義するための構文の一端も見ています。英数字の識別子で val や let を使うだけでなく、( = ) のような表記で英数字以外の記号からなる中置演算子を導入できます。詳細は こちら を参照してください。

命題的等値性

F* にはもう一つの等値概念、すなわち命題的等値性があり、== で表記します。任意の型 t に対し、項 e₁, e₂ : t が与えられたとき、命題 e₁ == e₂ は（決定不能でありうる）``e₁`` と e₂ の等しさを主張します。命題的等値演算子の型は次のとおりです。

val ( == ) (#a:Type) (x:a) (y:a) : prop

決定可能な等値 (=) と異なり、命題的等値はあらゆる型に対して定義されています。(==) の結果の型は prop、すなわち命題の型です。これについては 次章 でさらに学びます。
