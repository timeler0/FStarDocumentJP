プログラムの実行

これまでに複数の章を通して F* の中核となる概念を学んできましたが、プログラムのコンパイルと実行についてはまだ触れていません。ここまでは主に F* の論理と、プログラムに関する性質の証明に焦点を当ててきたからです。

F* にはプログラムを実行する方法がいくつかあります。ここでは Quicksort を通例の例として、概要を手短に紹介します。

F* プログラムのインタプリタ実行

カプセル要約 で述べたように、F* には *正規化器*（normalizer）と呼ばれる、計算を記号的に簡約するエンジンが備わっています。正規化器の活用方法は今後もたくさん登場しますが、基本的な使い方の一つがプログラムのインタプリタ実行です。

emacs の fstar-mode.el を使えばインタプリタの起動は簡単です。emacs の F* メニューから Interactive queries → Evaluate an expression を選ぶ（または C-c C-s C-e）と、評価したい式の入力を求められます。sort ( <= ) [4;3;2;1] と入力して Enter を押すと、sort ( <= ) [4;3;2;1] \(\downarrow\beta\delta\iota\zeta\) [1; 2; 3; 4] <: Prims.Tot (list int) のように表示され、与えた項が型 Tot (list int) を持つ [1; 2; 3; 4] に簡約されたことが分かります。

\(\downarrow\beta\delta\iota\zeta\) は一見難解に見えますが、項の解釈に用いた簡約戦略を示しています。

  • \(\beta\) は関数適用（β-簡約）を行ったことを意味します。

  • \(\delta\) は定義の展開（δ-簡約）を行ったことを意味します。

  • \(\iota\) はパターンマッチの実行（ι-簡約）を意味します。

  • \(\zeta\) は再帰関数の展開（ζ-簡約）を意味します。

これらの簡約段階の意味や、用途に応じたカスタマイズ方法は後の章で改めて説明します。

OCaml へのコンパイル

F* プログラムを実行する主な方法は、OCaml にコンパイル（抽出）し、OCaml のビルドシステムと実行環境で実行ファイルを作って動かすことです。

注釈

ここで紹介する手順は数ファイル程度のシンプルなプロジェクト向けです。大規模プロジェクトでは、Makefile 用の依存関係を生成できる依存解析や、ファイル単位で検査して結果をキャッシュ・再利用できる分割コンパイルを利用できます。これらの活用法や大規模ビルドの構成については以下を参照してください。

OCaml ライブラリの生成

F* プログラムから OCaml コードを抽出するには、以下のようにコマンドラインオプションを使います。

fstar --codegen OCaml --extract Part1.Quicksort --odir out Part1.Quicksort.Generic.fst

  • --codegen は OCaml コードの生成を指示します。

  • --extract は指定した名前空間内のモジュールのみを抽出する指定です。ここでは Part1.Quicksort 配下のすべてが対象です。

  • --odir は生成物の出力先ディレクトリを指定します（ここでは out）。

  • 最後の引数は検査と抽出の対象となるソースファイルです。

The resulting OCaml code is in the file Part1_Quicksort_Generic.ml, where the F* dot-separated name is transformed to OCaml's naming convention for modules. The generated code is here

抽出コードに関する注意点：

  • 抽出対象は実行可能コードに対応する定義のみです。補題など証明専用の部分は消去されます。消去（erasure）については後の章で詳しく扱います。

  • F* の型は OCaml の型に翻訳されます。F* の型は OCaml より精密なので、翻訳には不可避の情報落ちが伴います。例えば Part1.Quicksort.Generic.fst における全順序の型は次のとおりですが、

    let total_order_t (a:Type) = f:(a -> a -> bool) { total_order f }

    OCaml では次のようになります。

    type 'a total_order_t = 'a -> 'a -> Prims.bool

    つまり、未検証の OCaml コードから抽出した F* 関数を呼ぶ際は注意が必要です。F* 側が全順序を期待する箇所に、OCaml 側で全順序でない関数を渡してもコンパイラは警告してくれません。

OCaml ライブラリのビルド

抽出されたコードは（sort などの）トップレベル関数を提供しますが、main は含みません。したがってライブラリとしてのみビルドできます。

簡単な用途であれば、以下のように ocamlbuild を用いて、生成コードを OCaml のネイティブライブラリ（cmxa）にビルドできます。

OCAMLPATH=$FSTAR_HOME/lib ocamlbuild -use-ocamlfind -pkg batteries -pkg fstar.lib Part1_Quicksort_Generic.cmxa

注意点：

  • 必要なライブラリ探索に使われる OCAMLPATH を設定する必要があります。F* プロジェクトでは、F* リリースの bin ディレクトリを``OCAMLPATH`` に含めてください。

  • -use-ocamlfind は OCaml ライブラリ探索ユーティリティの利用を有効にします。

  • 抽出された F* プログラムは batteries と fstar.lib の 2 つのライブラリに依存します。-pkg オプションで指定しています。

  • 最後に、Part1_Quicksort_Generic.cmxa は対応する .ml の名前に .cmxa 拡張子を付けたもので、ライブラリとしてビルドすることを示します。

生成された .cmxa は他の OCaml プロジェクトから利用できます。

「main」を追加する

これまで扱ってきたのは 全 関数のプログラミングと証明でした。全関数は副作用を持ちません（状態の読み書きや出力などはできません）。これはライブラリには適していますが、出力を行うトップレベルのドライバ（main）を書くには、実際に副作用を持つ関数を書く必要があります。

副作用プログラムに対する F* の支援については後の章で詳しく学びます。ここではそのさわりとして、Quicksort ライブラリを呼び出す main プログラムを示します。

module Part1.Quicksort.Main
module Q = Part1.Quicksort.Generic

//Printing the elements of an integer list, using the FStar.Printf library
let string_of_int_list l =
  Printf.sprintf "[%s]"
    (String.concat "; " (List.Tot.map (Printf.sprintf "%d") l))

//A main program, which sorts a list and prints it before and after
let main () =
  let orig = [42; 17; 256; 94] in
  let sorted = Q.sort ( <= ) orig in
  let msg =
    Printf.sprintf "Original list = %s\nSorted list = %s\n"
      (string_of_int_list orig)
      (string_of_int_list sorted)
  in
  FStar.IO.print_string msg

//Run ``main ()`` when the module loads
#push-options "--warn_error -272"
let _ = main ()
#pop-options

ここでの注意点：

  • 今回は未検証の OCaml からではなく F* から Q.sort を呼ぶため、前提条件（渡す比較関数 ( <= ) が本当に全順序である等）を満たすことを証明する必要があります。これらは F* が自動で処理してくれます。

  • FStar.IO.print_string は文字列を stdout に出力するライブラリ関数です。型は string -> ML unit で、この ML ラベルを持つ型は、入出力や例外などの観測可能な副作用を伴う可能性があることを示します。詳細は効果の章で扱います。

  • ファイル末尾の let _ = main () は、実行時に（標準出力への印字という）副作用を持つトップレベル項です。複数モジュールがある場合、このようなトップレベル副作用の実行順はモジュール読み込み順に依存します。F* はこれを検出すると警告 272 を出します。本例では意図的にトップレベルの効果を持たせているため、--warn_error -272 で警告を抑制します。

このコードを Part1.Quicksort.Generic への依存とともに OCaml に抽出するには、次を実行します。

fstar --codegen OCaml --extract Part1.Quicksort --odir out Part1.Quicksort.Main.fst

今回は Part1.Quicksort.Generic.fst``（前と同様）に加えて ``Part1.Quicksort.Main.fst も OCaml に抽出され、Part1_Quicksort_Main.ml が生成されます。

この OCaml コードは、次のようにしてネイティブ実行ファイルへビルドできます。

OCAMLPATH=$FSTAR_HOME/lib ocamlbuild -use-ocamlfind -pkg batteries -pkg fstar.lib Part1_Quicksort_Main.native

最後に Part1_Quicksort_Main.native を実行すると、次のような出力が得られます。

$ ./Part1_Quicksort_Main.native
Original list = [42; 17; 256; 94]
Sorted list = [17; 42; 94; 256]
