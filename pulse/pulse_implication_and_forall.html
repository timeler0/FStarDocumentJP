含意と全称量化

この章では分離論理の連結子 @==> と forall* を紹介します。まずはごく簡単な例で使い方を示し、次章の連結リストではこれらの連結子のより本格的な用例を見ていきます。

Trade（分離ゴースト含意）

ライブラリ module I = Pulse.Lib.Trade.Util は trade 演算子 (@==>) と その補助関数群を提供します。文献では p --* q を “magic wand” と呼びますが、p @==> q はこれに近く、いくつかの技術的差異があります（後述）。ここでは「p を q にトレードする」と読みます。p @==> q の直観的意味は次のとおりです。

p @==> q は「p を持っていればそれを 交換 して q を得られる」ことを表します。すなわち p ** (p @==> q) から q を導けます。この推論は以下の型を持つゴースト関数 I.elim によって実行されます。

ghost
fn I.elim (p q:slprop)
requires p ** (p @==> q)
ensures q

重要なのは、p を資源に対する許可だと考えると、I.elim により p の許可を 手放し、その見返りとして q を得る点です。さらにこの際、含意自体の権利も消費されます。つまり p @==> q によるトレードは一度きりです。

では最初に p @==> q はどう作るのでしょうか。導入規則は次のとおりです。

ghost
fn I.intro (p q r:slprop)
           (elim: unit -> stt_ghost unit emp_inames (r ** p) (fun _ -> q))
requires r
ensures p @==> q

すなわち p @==> q を導入するには、ゴースト関数で r ** p を q に変換できるような補助の許可 r を保持していることを示します。

Share と Gather

p @==> q の動作を示す小さな例を見てみましょう。

let regain_half (#a: Type0) (x:GR.ref a) (v:a) =
  pts_to x #0.5R v @==> pts_to x v

述語 regain_half は、半分の許可 pts_to x #one_half v を全許可 pts_to x v にトレードできることを表します。一見直感に反しますが、p @==> q 自体が内部に権利を保持している点が肝心です。具体的には regain_half x v は内部に exists* u. pts_to x #one_half u を持ち、文脈がもう半分を提示すれば、除去子がそれらを合成して全許可を返せます。

regain_half の導入方法を見てみましょう。

ghost 
fn intro_regain_half (x:GR.ref int)
requires pts_to x 'v
ensures pts_to x #0.5R 'v ** regain_half x 'v
{
  GR.share x;
  intro (pts_to x #0.5R 'v @==> pts_to x 'v) #(pts_to x #0.5R 'v) fn _
  {
    GR.gather x;
  };
  fold regain_half;
}

仕様は、pts_to x 'v から始めれば、それを pts_to x #one_half v と regain_half x 'v に分割できると述べています。通常の分割では二つの半許可に分けますが、ここでは片方をゴースト関数に包み、必要時に再結合（gather）できるようにしています。

実装では、pts_to x #one_half 'v @==> pts_to x 'v の除去に相当する補助ゴースト関数を定義します（中身は gather）。その後 pts_to x 'v を半分に分け、除去子を渡して I.intro を呼び、regain_half に畳み込みます。要するに regain_half は、補助ゴースト関数 aux と x の半許可をひとかたまりの slprop にしただけです。

後で regain_half を使いたいときは、その除去子を呼びます――実質的には必要な許可を渡して aux を呼ぶのと同じです。

ghost
fn use_regain_half (x:GR.ref int)
requires pts_to x #0.5R 'v ** regain_half x 'v
ensures pts_to x 'v
{
  unfold regain_half;
  I.elim _ _;
}

ここまで来ると、「最初から pts_to x #one_half 'v と gather で良いのでは？」と思うかもしれません。この単純な例では確かに (@==>) の利得は大きくありません。

全称量化

改めて regain_half を見直します。

let regain_half (#a: Type0) (x:GR.ref a) (v:a) =
  pts_to x #0.5R v @==> pts_to x v

この述語は汎用性に欠けます。除去には、導入時と 同じ v に対する pts_to x #one_half v を持っていることを呼び出し側が示さねばなりません。

やや一般化して次のようにもできます。

let regain_half #a (x:GR.ref a) (v:a) =
  (exists* u. pts_to x #one_half u) @==> pts_to x v

改善ではありますが、存在量化された u と v の関係が表せず、まだ不十分です。必要なのは全称量化です。

正しい形は次のとおりです。

let regain_half_q (#a: Type0) (x:GR.ref a) =
  forall* u. pts_to x #0.5R u @==> pts_to x u

これは、文脈がどの pts_to x #one_half u を持っていても、同じ証人 u のまま全許可に戻せる、という主張になります。

forall* とその操作用ユーティリティは Pulse.Lib.Forall.Util にあります。導入・除去の形は @==> とよく似ています。

ghost
fn FA.elim (#a:Type) (#p:a->slprop) (v:a)
requires (forall* x. p x)
ensures p v

除去子は全称変数 x を v に 一回だけ インスタンス化することを許します。

ghost
fn FA.intro (#a:Type) (#p:a->slprop)
     (v:slprop)
     (f_elim : (x:a -> stt_ghost unit emp_inames v (fun _ -> p x)))
requires v
ensures (forall* x. p x)

導入には、v を保持しており、任意の x に対してゴースト関数で v から p x を生み出せることの証明が要ります。

全称量化と trade を組み合わせるのは一般的なので、ライブラリには次のような合成版も用意されています。

ghost
fn elim_forall_imp (#a:Type0) (p q: a -> slprop) (x:a)
requires (forall* x. p x @==> q x) ** p x
ensures q x

および

ghost
fn intro_forall_imp (#a:Type0) (p q: a -> slprop) (r:slprop)
         (elim: (u:a -> stt_ghost unit emp_inames
                          (r ** p u)
                          (fun _ -> q u)))
requires r
ensures forall* x. p x @==> q x

Share と Gather（再び）

regain_half_q の導入は次のとおりです。

ghost 
fn intro_regain_half_q (x:GR.ref int)
requires pts_to x 'v
ensures pts_to x #0.5R 'v ** regain_half_q x
{
  GR.share x;
  intro (forall* u. pts_to x #0.5R u @==> pts_to x u)
      #(pts_to x #0.5R 'v) fn _ u {
    gather x;
  };
  fold regain_half_q;
}

これで利用時には、任意の pts_to x #one_half u``（の半許可）を、同じ ``u の全許可にトレードできます。

ghost
fn use_regain_half_q (x:GR.ref int)
requires pts_to x #0.5R 'u ** regain_half_q x
ensures pts_to x 'u
{
  unfold regain_half_q;
  FA.elim #_ #(fun u -> pts_to x #0.5R u @==> pts_to x u) 'u;
  I.elim _ _;
}

FA.elim の除去子は少々冗長で、量化子の項を再度明示する必要があります。現在の Pulse における F* のユニファの使い方では、高階単一化の一部にうまく解が見つからないことがあるためです。これは今後改善予定です。

Trade とゴーストステップ

最後に、ゴースト状態を変更しうるステップも含め、任意のゴースト計算を trade に包めることを示します。一般には、@==> は “wand” というより Iris 用語でいう view shift に近い振る舞いになります。

述語 can_update は、pts_to x #one_half u の半許可を、別の値 pts_to x v の全許可へとトレードできることを表します。

let can_update (x:GR.ref int) = 
  forall* u v. pts_to x #0.5R u @==>
               pts_to x v

make_can_update では、ゴースト状態の更新関数を二変数の量化 forall* u v. ... にパッケージします。

ghost
fn make_can_update (x:GR.ref int)
requires pts_to x 'w
ensures pts_to x #0.5R 'w ** can_update x
{
  share x;
  intro (forall* u v. pts_to x #0.5R u @==> pts_to x v)
      #(pts_to x #0.5R 'w) fn _ u v {
    gather x;
    x := v;
  };
  fold (can_update x);
}

以下の update では、それを具体化して参照 x を 'u から k に更新し、さらに呼び出し側が再利用できるよう can_update 述語も返します。

ghost
fn update (x:GR.ref int) (k:int)
requires pts_to x #0.5R 'u ** can_update x
ensures pts_to x #0.5R k ** can_update x
{
  unfold can_update;
  FA.elim #_ #(fun u -> forall* v. pts_to x #0.5R u @==> pts_to x v) 'u;
  FA.elim #_ #(fun v -> pts_to x #0.5R 'u @==> pts_to x v) k;
  I.elim _ _;
  make_can_update x;
}
