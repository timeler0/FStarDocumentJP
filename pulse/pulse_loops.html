ループと再帰

この章では、while をはじめとするさまざまなループ構文の動作と、再帰関数を Pulse でどのように扱うかを見ていきます。

既定では、Pulse のロジックは部分正当性を対象としています。これは、プログラムが無限にループしてもよいことを意味します。プログラムが事後条件 p を満たす v:t を返すと言うとき、それは「実行が無限ループする可能性はあるが、もし返るならそのときの状態は必ず p v を満たす」という意味です。

while ループの一般形

while ループの形は次のとおりです。

while ( guard )
invariant (b:bool). p
{ body }

ここで

  • guard は b:bool を返す Pulse プログラムです。

  • body は unit を返す Pulse プログラムです。

  • invariant (b:bool). p は次を満たす不変条件です。

    • ループ突入前および各 body の事後条件として exists* b. p が証明可能でなければなりません。

    • exists* b. p はガードの事前条件、p b はその事後条件です。すなわち guard は次を満たす必要があります。

      requires exists* b. p
returns b:bool
ensures p

    • ループ全体の事後条件は invariant false です。

この不変条件は、3 つの異なるプログラム位置での主張をまとめて表すものだと理解できます。

  • b==true のとき、不変条件はループ本体の開始時点の状態を表します。

  • b==false のとき、不変条件はループ終了時点の状態を表します。

  • b が未確定のとき、不変条件はガードが（再）実行される直前、すなわちループ入口と本体末尾での状態の性質を表します。

ループを記述する不変条件を考えるには注意深い思考が必要になることが多いです。以降の章で多くの例を見ていきますが、ここでは簡単なループから始めます。

カウントダウン

最初のループ付き Pulse プログラムです。count_down は参照を 0 になるまで繰り返し減算します。

fn count_down (x:ref nat)
requires R.pts_to x 'v
ensures  R.pts_to x 0
{
    let mut keep_going = true;
    while (
        !keep_going
    )
    invariant
      exists* (b:bool) (v:nat).
        pts_to keep_going b **
        pts_to x v **
        pure (not b  ==> v == 0)
    {
        let n = !x;
        if (n = 0) 
        {
            keep_going := false;
        } 
        else
        {
            x := n - 1;
        }
    }
}

Pulse の while ループは他言語より少し一般的です。guard は単にローカル変数を読むだけでなく、任意の Pulse プログラムにできます。例えば次の版では、guard がすべての処理を行い、ループ本体は空、補助変数 keep_going も不要です。

fn count_down3 (x:ref nat)
requires R.pts_to x 'v
ensures  R.pts_to x 0
{
    while (
        let n = !x;
        if (n = 0)
        {
            false
        } 
        else
        {
            x := n - 1;
            true
        }
    )
    invariant
      exists* (v:nat). pts_to x v
    { () }
}

部分正当性

部分正当性の解釈により、次のような無限ループする変種も受理されます。

fn count_down_loopy (x:ref nat)
requires R.pts_to x 'v
ensures  R.pts_to x 0
{
    while (
        let n = !x;
        if (n = 0)
        {
            false
        }
        else
        {
            x := n + 1;
            true
        }
    )
    invariant exists* v. R.pts_to x v
    { () }
}

このプログラムは x を減らす代わりに増やしますが、常に無限ループするため、以前と同じ不変条件を満たします。

ただし Pulse には、ghost と atomic 計算のロジックなど、必ず停止する部分があります。将来的には、汎用の逐次プログラム（並行なし）向けにも停止性を保証する Pulse ロジックの版を提供する予定です。

加算の繰り返しによる乗算

次のループプログラムは、自然数 x, y に対して、加算を x 回繰り返して積を求めます。この題材には歴史があります。Alan Turing の 1949 年の論文 "Checking a Large Routine" は、コンピュータプログラムの正しさ証明に関する最初の論文としてしばしば引用されます。Turing が扱ったのも、この加算の繰り返しによる乗算でした。

fn multiply_by_repeated_addition (x y:nat)
    requires emp
    returns z:nat
    ensures pure (z == x * y)
{
    let mut ctr : nat = 0;
    let mut acc : nat = 0;
    while ( (!ctr < x) )
    invariant 
    exists* (c a : nat).
        R.pts_to ctr c **
        R.pts_to acc a **
        pure (c <= x /\
              a == (c * y))
    {
        let a = !acc;
        acc := a + y;
        let c = !ctr;
        ctr := c + 1;
    };
    !acc
}

いくつかの注目点：

  • カウンタ ctr とアキュムレータ acc はともに nat と宣言されており、精密型付けにより両者が常に 0 以上であるという不変条件が暗黙に与えられます。これは、Pulse が F* の依存型システムの上に分離ロジックを重ねて提供していることを示します。

  • 不変条件は、(1) カウンタは x を超えない、(2) アキュムレータは常にカウンタと y の積である、(3) カウンタが x より小さい間ループを続ける、という内容です。

最初の N 個の数の総和

次の例は、最初の n 個の自然数の合計を計算する Pulse プログラムです。純粋な F* の仕様や補題と併せて Pulse プログラムを開発する流れを示します。

まず、単純な再帰関数 sum の F* による仕様と、この和に関するよく知られた恒等式を示す補題から始めます。

let rec sum (n:nat)
: nat
= if n = 0 then 0 else n + sum (n - 1)

#push-options "--z3rlimit 20"
noextract
let rec sum_lemma (n:nat)
: Lemma (sum n == n * (n + 1) / 2)
= if n = 0 then ()
  else sum_lemma (n - 1)
#pop-options

次に、反復版の sum である isum を実装し、sum_lemma で示した恒等式を満たすことを証明します。

#push-options "--z3cliopt 'smt.arith.nl=false'"
noextract

fn isum (n:nat)
requires emp
returns z:nat
ensures pure ((n * (n + 1) / 2) == z)
{
    let mut acc : nat = 0;
    let mut ctr : nat = 0;
    while ( (!ctr < n ) )
    invariant 
    exists* (c a : nat).
        R.pts_to ctr c **
        R.pts_to acc a **
        pure (c <= n /\
              a == sum c)
    {
        let a = !acc;
        let c = !ctr;
        ctr := c + 1;
        acc := a + c + 1;
    };
    sum_lemma n; //call an F* lemma inside Pulse
    !acc;
}
#pop-options

このプログラムは multiply_by_repeated_addition によく似ていますが、いくつか違いがあります。

  • 不変条件は、アキュムレータの現在値が最初の c 個の総和であることを述べています。つまり、非線形算術の性質に頼ることなく（Z3 の非線形算術はプラグマで無効化しています）、このループが sum の再帰実装を精密化していることを示します。

  • 最後に、目的の恒等式の証明は、Pulse から既に証明済みの F* の sum_lemma を呼び出すだけで完了します。

このプログラム自体は少し作為的ですが、まず純粋な F* 関数の精密化として Pulse プログラムを示し、次にその純粋関数に対する数学的推論を用いて Pulse 側の性質を結論づけられる、という流れを示せたはずです。

再帰

Pulse は一般再帰、すなわち停止しない可能性のある再帰関数もサポートします。以下は簡単な例で、後ほどさらに例を見ます。

まず、n 番目のフィボナッチ数を計算する標準的な F* の（相互）再帰定義から始めます。

let rec fib (n:nat) : nat =
  if n <= 1 then 1
  else fib (n - 1) + fib (n - 2)

これを Pulse でも fib_rec として実装できます。直近 2 つのフィボナッチ数の値を保持するため、アウトパラメータを用います。

fn rec fib_rec (n:pos) (out:ref (nat & nat))
requires
    pts_to out 'v
ensures
    exists* v.
        pts_to out v **
        pure (
          fst v == fib (n - 1) /\
          snd v == fib n 
        )
{
  if (n = 1)
  {
    //type inference in Pulse doesn't work well here:
    //it picks (1, 1) to have type (int & int)
    //so we have to annotate
    out := ((1 <: nat), (1 <: nat)); 
  }
  else
  {
    fib_rec (n - 1) out;
    let v = !out;
    out := (snd v, fst v + snd v);
  }
}

ここでの注意点：

  • Pulse での再帰定義は fn rec で導入します。

  • fib の直近 2 値を容易にメモ化できるよう、引数 n は 0 を含めず正の数であることを仮定します。

  • コメントでも触れますが、(1, 1) を自然数の組として正しく型付けするには追加の型注釈が必要です。

もちろん、以下のように while ループで反復実装することも可能です。

fn fib_loop (k:pos)
  requires emp
  returns r:nat
  ensures pure (r == fib k)
{
  let mut i : nat = 1;
  let mut j : nat = 1;
  let mut ctr : nat = 1;
  while ((!ctr < k))
  invariant
    exists* (vi vj vctr : nat).
        R.pts_to i vi **
        R.pts_to j vj **
        R.pts_to ctr vctr **
        pure (
            1 <= vctr /\
            vctr <= k /\
            vi == fib (vctr - 1) /\
            vj == fib vctr) 
  {
      let vi = !i;
      let vj = !j;
      let c = !ctr;
      ctr := c + 1;
      i := vj;
      j := vi + vj;
  };
  !j
}
