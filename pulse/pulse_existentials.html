存在量化

Pulse でよく使われる仕様記述スタイルのひとつに、存在量化の利用があります。本格的な例に入る前に、存在量化がどのように機能するかを簡単に見ておきましょう。

Pulse の導入 で述べたとおり、Pulse の分離論理の連結子の1つが存在量化です。構文は F* の存在量化に似ていますが、exists ではなく exists* と書き、その本体は以下の例のように ``slprop``（分離論理命題）になります。

exists* (v:nat). pts_to x v

exists* v. pts_to x v

exists* v1 v2. pts_to x v1 ** pts_to y v2

...

いくつかの簡単な例

前章の assign の例（下記再掲）を振り返ると、事前条件で論理変数 'v をわざわざ束縛しているのはなぜか、と思われたかもしれません。実際には他の述語で使っていないからです。

fn assign u#a (#a:Type u#a) (r:ref a) (v:a)
requires pts_to r 'v
ensures pts_to r v
{
    r := v;
}

実際、論理変数の引数を使わずに assign の仕様を書く別の方法が以下です。

fn assign (#a: Type0) (x:ref a) (v:a)
requires
  exists* w. pts_to x w
ensures
  pts_to x v
{
  x := v
}

ここでは事前条件に存在量化を用い、x が任意の値 w を指している文脈であれば assign を呼び出せる、ということを表現しています。

しかし通常は、関数の事後条件は呼び出し前の初期状態と呼び出し後の状態を*関係付け*ます。存在変数の有効範囲は、囲んでいる slprop のできるだけ右側に限られます。そのため、関数の事前条件で存在量化を使うのはあまり一般的ではありません。

例として、次の incr の仕様はうまくいきません。存在量化で束縛した w0 が事後条件のスコープに入らないためです。

[@@expect_failure] 
fn incr #a (x:ref int)
requires
  exists* w0. pts_to x w
ensures 
  pts_to x (w0 + 1) //w0 is not in scope here
{
  let w = !x
  x := w + 1
}

一方で、存在量化は事後条件ではよく使われます。関数の振る舞いをあえて詳細化しすぎず、抽象化するためです。例えば下の make_even を見てください。この型は、x の内容を何らかの偶数 w1 に設定することだけを述べ、w1 の具体値は指定しません。また事後条件が x の初期値に依存しないため、事前条件にも存在量化を使っています。

fn make_even (x:ref int)
requires
  exists* w0. pts_to x w0
ensures
  exists* w1. pts_to x w1 ** pure (w1 % 2 == 0)
{
  let v = !x;
  x := v + v;
}

存在量化の操作

古典的連結子の扱い の章で、F* が存在量化を含む論理連結子の導入／除去のためのさまざまな構成子を提供していることを見ました。Pulse にも存在量化を明示的に扱う構成子があります。通常は Pulse の自動化が裏で導入・除去を行いますが、明示的操作が有用な場面もあります。以下でその最初の例を示します。

fn make_even_explicit (x:ref int)
requires
  exists* w0. pts_to x w0
ensures
  exists* w1. pts_to x w1 ** pure (w1 % 2 == 0)
{
  with w0. assert (pts_to x w0);
  let v = !x; // v:int; _:squash(v==w0); Ctxt=pts_to x v
  x := v + v; //                          ... pts_to x (v + v)
  introduce
  exists* w1. pts_to x w1 ** pure (w1 % 2 == 0)
  with (v + v);
}

存在量化の除去（消去）

with w0...wn. assert p; rest という形は、存在量化の除去子としてよく使われます。文脈に exists* x0...xn. p があるとき、with は以降のスコープ rest で 存在量化の変数に対応する証人として w0 ... wn を束縛します。

with w0. assert (pts_to x w0) の直後に show_proof_state を呼ぶと、次のように表示されます。

- Current context:
    pts_to x (reveal w0) **
    emp
- In typing environment:
    [w0#2 : erased int,
    x#1 : ref int]

すなわち、スコープ内に w0:erased int があり、文脈には pts_to x (reveal w0) が存在することがわかります。

with の別の例です。今度は複数の束縛変数を扱います。

fn make_even_explicit_alt (x y:ref int)
requires
  exists* wx wy. pts_to x wx ** pts_to y wy ** pure (wx % 2 == wy % 2)
ensures
  exists* wx' wy'. pts_to x wx' ** pts_to y wy' ** pure (wx' % 2 == 0)
{
  let vx = !x; 
  let vy = !y;
  x := vx + vy;
  introduce exists* wx' wy'. pts_to x wx' ** pts_to y wy' ** pure (wx' % 2 == 0)
  with (vx + vy) vy;
}

文脈に存在式がひとつだけある場合、with x1..xn. _ と書いてその式を「開き」、証人をスコープに束縛できます。先頭行の直後で show_proof_state を呼ぶと次のように表示されます。

- Current context:
    pts_to x (reveal wx) **
    pts_to y (reveal wy) **
    pure (eq2 (op_Modulus (reveal wx) 2) (op_Modulus (reveal wy) 2)) **
    emp
- In typing environment:
    [_#5 : squash (eq2 (op_Modulus (reveal wx) 2) (op_Modulus (reveal wy) 2)),
    wy#4 : erased int,
    wx#3 : erased int,
    y#2 : ref int,
    x#1 : ref int]

存在量化の導入

Pulse のチェッカは、存在量化の各変数に対して新しい単一化変数を導入し、ゴール側と文脈側の slprop の照合によってその解を探すことで、存在式を自動で導入します。

ただし、上の 2 つの例のように、introduce exists* 構文を用いて存在式を明示的に導入することもできます。一般に次のように書けます。

introduce exists* x1 .. xn. p
with w1...wn

ここで、存在量化された各変数 x1..xn に対する証人 w1..wn を明示的に与えます。
