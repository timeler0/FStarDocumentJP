Pulse: 並行分離論理における証明指向プログラミング

多くの F* プロジェクトでは、特殊化されたプログラミングおよび証明サポートを備えたドメイン固有言語（DSL）を構築します。例えば、Vale は構造化アセンブリ言語のプログラム証明を、Low* は C 風メモリモデルの副作用的プログラミングを、EverParse は低レベルのパーサ／シリアライザ記述用 DSL を提供します。近年 F* は、Pulse をショーケースとするような、F* に埋め込まれた DSL を構築するためのカスタム構文・型チェッカプラグイン・抽出サポートなどの新機能を獲得しました。

Pulse は F* に埋め込まれた新しいプログラミング言語で、（高階性や依存型など）F* の多くの機能を受け継ぎつつ、可変状態と並行性を伴うプログラミングを、並行分離論理 による仕様と証明とともに標準でサポートします。

まずは Pulse の味見として、可変な整数参照をインクリメントする関数を示します。

fn incr (x:ref int)
requires pts_to x 'i
ensures pts_to x ('i + 1)
{
    let v = !x;
    x := v + 1;
}

そして、2 つの参照を並行にインクリメントする関数がこちらです。

fn par_incr (x y:ref int)
requires pts_to x 'i ** pts_to y 'j
ensures pts_to x ('i + 1) ** pts_to y ('j + 1)
{
   par (fun _ -> incr x)
       (fun _ -> incr y)
}

分離論理に馴染みがないかもしれませんが、これらの仕様は直感的に理解できるはずです。incr の型は、初期状態で「x が 'i を指している」なら、incr の返却時には「x が 'i + 1 を指している」と述べています。一方 par_incr は par コンビネータを用いて x と y の内容を並行にインクリメントします。

並行分離論理は活発な研究分野であり、さまざまな論理が提案され、それぞれにトレードオフがあります。最先端は Iris で、高階かつ非正則的（impredicative）な分離論理です。Pulse は Iris から着想を得て多くの点で類似していますが、F* の内部だけで形式化された PulseCore という論理に基づいています――形式化は こちら にあります。Pulse の表面言語でのプログラムの証明は、PulseCore のプログラム論理での正しさの証明に対応します。ただし、Pulse を有効に使うために形式化の詳細を深く知る必要はありません。基礎から始めて、Pulse でプログラミングと証明を行うのに必要な並行分離論理の要点を説明します。また、Pulse は F* の拡張なので、F*、補題、依存型、篩型（refinement types）などについて学んだことはそのまま役立ちます。

注釈

なぜ “Pulse” という名前なのでしょう？ それは、以前の論理 Steel から発展したものであり、著者の一人とその娘がクラシックなレゲエバンド Steel Pulse の大ファンだからです。Steel よりも柔らかく、少し遊び心のある名前にしたかった――それで “Pulse” になりました。
