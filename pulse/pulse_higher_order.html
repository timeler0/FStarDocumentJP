高階関数

F* と同様に、Pulse も高階です。つまり、Pulse の関数は第一級であり、他の関数に引数として渡したり、結果として返したり、ものによってはヒープに格納することもできます。

Pulse の計算型

おそらく最も単純な高階関数の例として、関数適用を抽象化する apply を示します。

fn apply (#a:Type0)
         (#b:a -> Type0)
         (#pre:a -> slprop)
         (#post: (x:a -> b x -> slprop))
         (f: (x:a -> stt (b x) (pre x) (fun y -> post x y)))
         (x:a)
requires pre x
returns y:b x
ensures post x y
{
  f x
}

この関数は、引数 x に適用する関数 f の引数型・結果型・前提・事後条件に関して多相です。ここでは初めて、Pulse 関数の型を F* の型として記述します。これまでは fn/requires/ensures 記法で Pulse 関数の*シグネチャ*を書いてきましたが、Pulse 関数の型は次の形になります。

x:a -> stt b pre (fun y -> post)

ここで、

  • 他の F* の関数型と同様、Pulse の関数も依存型であり、矢印の右側（戻り側）の型に x を参照できます。

  • 矢印の直後には Pulse の計算型タグが来ます。これは F* の Tot や GTot などに相当します。

  • stt は Pulse の計算型タグのうち最も許容的で、関数本体が状態の読み書きや無限ループなどを行えます。ただし前提は pre、戻り値の型は b、事後条件は fun y -> post です。

Pulse には他にもいくつかの計算型があります。ここで重要なのはゴースト計算用のコンストラクタです。以下に、apply のゴースト版である apply_ghost を示します。

ghost
fn apply_ghost 
         (#a:Type0)
         (#b:a -> Type0)
         (#pre:a -> slprop)
         (#post: (x:a -> b x -> slprop))
         (f: (x:a -> stt_ghost (b x) emp_inames (pre x) (fun y -> post x y)))
         (x:a)
requires pre x
returns y:b x
ensures post x y
{
  f x
}

f の型は先ほどと似ていますが、今回は次の相違があります。

  • 計算型タグが stt_ghost であり、ゴースト状態のみを読み書きし、必ず停止することを表します。

  • 戻り値の型が b x であること

  • 次の引数 emp_inames は、その計算が開く可能性のある不変量集合を表し、emp_inames はいかなる不変量も開かないことを意味します。ここでは詳細は無視して構いません。

  • 前提は pre x、事後条件は fun y -> post x y です。

ユニバース

参考までに、stt と stt_ghost のシグネチャを以下に示します。

val stt (a:Type u#a) (i:inames) (pre:slprop) (post: a -> slprop)
  : Type u#0

val stt_ghost (a:Type u#a) (i:inames) (pre:slprop) (post: a -> slprop)
  : Type u#4

注意点として、stt 計算はユニバース u#0 に属します。stt は無限ループを許し、Div の上に構築されており、この Div も u#0 に属するからです。stt のユニバースにより、stt 関数は参照に格納できます（例：ref (unit -> stt unit p q) は Pulse で正当な型です）。

対照的に、stt_ghost 関数は全て停止し、ユニバース 4 に属します。もし stt_ghost 関数を状態に格納できてしまうと、stt_ghost の内部で停止しない関数を書けてしまうため、格納はできません。

カウンタ

高階プログラミングのもう少し興味深い応用として、可変カウンタの実装を見てみましょう。まずカウンタの型 ctr から定義します。

noeq
type ctr = {
    inv: int -> slprop;
    next: i:erased int -> stt int (inv i) (fun y -> inv (i + 1) ** pure (y == reveal i));
    destroy: i:erased int -> stt unit (inv i) (fun _ -> emp)
}

カウンタは次の要素をまとめたものです。

  • 状態上の述語 inv。inv i は「現在のカウンタ値が i である」ことを表し、実装（どのように状態を表現しているか）には立ち入りません。

  • 状態付き関数 next。前提として inv i を受け取り、現在値 i を返し、事後に inv (i + 1) を提供します。

  • カウンタを解放するための状態付き関数 destroy。

ctr を実装する一つの方法は、状態をヒープ上の参照で表すことです。以下の new_counter はまさにそれを行います。

fn new_counter ()
requires emp
returns c:ctr
ensures c.inv 0
{
    open Pulse.Lib.Box;
    let x = alloc 0;
    fn next (i:erased int)
    requires pts_to x i 
    returns j:int
    ensures pts_to x (i + 1) ** pure (j == reveal i)
    {
        let j = !x;
        x := j + 1;
        j
    };
    fn destroy (i:erased int)
    requires pts_to x i
    ensures emp
    {
       free x
    };
    let c = { inv = pts_to x; next; destroy };
    rewrite (pts_to x 0) as (c.inv 0);
    c
}

動作は次のとおりです。

まず、0 で初期化した新しいヒープ参照 x を確保します。

Pulse では任意のスコープ内で関数を定義できます。そこで、局所関数 next と destroy を定義します。実装と仕様は素直で、重要なのはこれらの関数がクロージャ内で x:box int を捕捉する点です。

最後に、next と destroy を c:ctr にまとめ、inv を Box.pts_to x に具体化し、コンテキストの主張を c.inv 0 に書き換えて c を返します。

呼び出し側（以下の test_counter など）からは、カウンタが単一の可変ヒープ参照で実装されている事実は完全に隠蔽されています。

fn test_counter ()
requires emp
ensures emp
{
    let c = new_counter ();
    let next = c.next;
    let destroy = c.destroy;
    let x = next _; //FIXME: Should be able to write c.next
    assert pure (x == 0);
    let x = next _;
    assert pure (x == 1);
    destroy _;
}
