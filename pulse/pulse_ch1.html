Pulse の基本

F* のファイルにディレクティブ #lang-pulse を記述すると、Pulse プログラムを F* に埋め込めます。以降の部分では、下記のように Pulse と F* の構文を混ぜて使用できます。

#lang-pulse

let fstar_five : int = 5

fn five ()
  requires emp
  returns n:int
  ensures pure (n == 5)
{ 
  fstar_five
}


let pulse_five_in_fstar = five ()

このプログラムは、まず通常の F* で fstar_five を定義し、続いて Pulse の関数 five を定義します。five はその F* の定義を参照し、常に定数 5 を返すことを証明します。最後に、Pulse で定義した five を通常の F* から参照しています。とても単純ですが、Pulse と F* が双方向にやり取りできることを示しています。

以下では、コード片が F* の大きな文脈に埋め込まれた Pulse であることを特に強調しない限り、#lang-pulse が有効な文脈で作業しているものとします。

分離論理（Separation Logic）入門

分離論理は 1990 年代後半、John Reynolds、Peter O'Hearn らによって、共有可能で可変なデータ構造（例：連結リストやグラフ）を用いる命令型プログラムを扱うために提案されました。概要は「an introduction to separation logic」をご参照ください。その後の数十年で多くの発展が加わり、ヒープ操作を行う逐次プログラムのみならず、時間・空間、通信メッセージなど多様な抽象資源を扱う分散・並行・非同期プログラムへと一般化されました。

前章 で見た他のホーア論理と同様に、分離論理も大きく二つの要素から成ります。

分離論理の命題（slprop） まず、プログラム資源（例：ヒープ）に関する性質を表す命題の言語があります。Pulse ではこれを slprop 型で表します。PulseCore の意味論では、slprop = state -> prop と見なされ、state はメモリ内容などプログラム状態を表します。初めは slprop を「メモリの性質」と捉えると理解しやすいですが、最終的にはより抽象的に扱い、多様な資源のモデル化に使います。

分離論理のホーア三つ組 slprop をプログラムと結び付けるため、分離論理ではホーア三つ組を用いてプログラムが状態に与える作用を記述します。例えば { p } c { n. q } は、前提 p s0 を満たす初期状態 s0 でプログラム c を走らせると、値 n を返しつつ状態を q n s1 を満たす s1 に変換することを表します。Pulse のプログラム論理は部分正当性であり、c が無限ループやデッドロックに陥る可能性は排除しません。

簡単な slprop と三つ組：もっとも基本的な例を二つ挙げます。

（Pulse.Lib.Pervasives で定義）

  • emp：自明な命題（fun s -> True と同等）。

  • pure p：ヒープに依存しない述語 fun s -> p。emp は pure True に等しい。

プログラム five の型は、これらの slprop が仕様でどのように使われるかを示しています。

  • 引数は unit ひとつの関数です——Pulse の関数はキーワード fn を使います。

  • 前提は emp``（分離論理における自明な主張）だけなので、``five はどんな初期状態からでも呼び出せます。

  • 戻り値は整数 n:int です。

  • 事後条件では戻り値の名前（ここでは n）を参照でき、最終状態で pure (n == 5) が成り立つと述べています。

言い換えると、Pulse の型注釈は { emp } five () { n:int. pure (n == 5) } というホーア三つ組を簡潔に書く方法です。

所有権（Ownership） ここで five の事後条件が十分強いか疑問に思うかもしれません。戻り値が n == 5 であることしか述べておらず、呼び出し後の状態については何も言っていません。pure (5 == 5) は任意の状態で真なので、five が状態のあらゆる場所を勝手に変更できてしまうのでは？という懸念です。Low* や Dafny などホーア論理系の言語に慣れている方は、どの領域を変更し得るかを記す modifies 節が無いことに驚くかもしれません。分離論理の良い点は、どの部分を変更したかを列挙する必要がないことです。中心概念が 所有権 だからです。第一近似として、計算は前提で明示的に与えられた資源、または自身で新たに作成した資源にしかアクセスできません。この例では前提が emp なので、five は いかなる 資源にもアクセスする権限がなく、従って観測可能な形で状態を変更できません。

分離積（Separating Conjunction）とフレーム規則 先の節で見た incr と par_incr の型を詳しく見直します。そのために、よく使う slprop を二つ導入します。まずは「ポインツ・トゥ（points-to）」述語です。

  • pts_to x v は、参照 x が現在の状態で値 v を保持するセルを指していることを主張します。

slprop はさまざまな方法で結合できます。最も一般的なのが「分離積」で、Pulse では ** と記します。 [3]

  • p ** q は、状態を互いに 素な 二つの部分に分割でき、それぞれが p と q を満たすことを意味します。言い換えれば、与えられた状態で p と q に対応する権限をそれぞれ保持している、という読み方もできます。** は次の法則を満たします。

    • 可換律：p ** q は q ** p に同値

    • 結合法則：p ** (q ** r) は (p ** q) ** r に同値

    • 単位元：p ** emp は p に同値（可換なので emp ** p も p に同値）

そして分離論理を特徴付けるのが、この ** がプログラム論理で果たす役割、すなわち フレーム規則 です。この規則は、もし { p } c { n. q } を示せるなら、任意の f : slprop に対して { p ** f } c { n. q ** f }``も示せる、と述べます。ここで ``f は「フレーム」と呼ばれます。直観になじむには時間がかかるかもしれませんが、フレーム規則は局所・モジュール的推論の本質を捉えています。すなわち、入力状態に対して p という権限だけを持つときに正しいプログラムは、より大きな状態で実行しても正しく、触れない部分（f）に関する性質を保つ、ということです。

これを踏まえて、x に対する権限だけを要求してそれをインクリメントする incr の型をもう一度見てみましょう。

fn incr (x:ref int)
requires pts_to x 'i
ensures pts_to x ('i + 1)
{
    let v = !x;
    x := v + 1;
}

フレーム規則により、以下の incr_frame のような文脈でも incr を呼べ、追加の作業なしに y が変わらないと示せます。

fn incr_frame (x y:ref int)
requires pts_to x 'i ** pts_to y 'j
ensures pts_to x ('i + 1) ** pts_to y 'j
{
   incr x;
}

実際、Pulse では任意の f:slprop に対してフレーム規則を適用でき、incr x が f を乱さないことをタダで得られます。

fn incr_frame_any (x:ref int) (f:slprop)
requires pts_to x 'i ** f
ensures pts_to x ('i + 1) ** f
{
   incr x;
}

記法について：変数 'i は暗黙に束縛される論理変数で、初期状態における参照 x の中身の値を表します。ここで 'i は型 FStar.Ghost.erased int を持ちます——以前 に学んだ消去型です。論理変数は明示的に束縛することもでき、例えば次と同等です。

fn incr_explicit_i (x:ref int) (i:erased int)
requires pts_to x i
ensures pts_to x (i + 1)
{
    let v = !x;
    x := v + 1;
}

その他の slprop 結合子 分離積以外にも、Pulse（および他の分離論理）には slprop を組み合わせる方法があります。詳細は後続章で扱いますが、雰囲気をつかむためによく使うものを挙げます。

  • exists* (x1:t1) ... (xn:tn). p：存在量化。Pulse のライブラリでは多用され、使いやすくする道具立ても豊富です。状態 s で、ある証人 w に対し p[w/x] が成り立つなら exists x. p は成り立ちます。上級者向け注意：存在量化は非述語的であり、slprop 自身に量化できます（例：exists* (p:slprop). q）。

  • forall* (x1:t1) ... (xn:tn). p：全称量化。使用頻度は高くありませんがサポートされています。すべての w:t に対して p[w/x] が成り立つなら、状態 s で forall (x:t). p は成り立ちます。存在量化と同様に非述語的です。

  • p @==> q は分離含意の一種で、他の分離論理で magic wand や view shift と呼ばれる演算子に相当します。

Pulse には（論理的な）連言・選言のためのライブラリはまだありません。とはいえ Pulse は F* に埋め込まれているため、ユーザーが新しい slprop を定義できます。再帰的述語や、上記結合子の変種を自作するのは一般的です。

[1]

上級者向けメモ：Pulse の分離論理は アフィン（affine） です。

[2]

後で不変条件やロックを扱う際、権限を別の手段で獲得する方法も見ていきます。

[3]

分離論理の文献では分離積は通常 p * q``（アスタリスク 1 個）で書かれますが、乗算との衝突を避けるため、本書では ``** を用います。
