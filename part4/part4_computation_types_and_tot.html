Total な計算の効果

効果ラベル階層の最下層には Tot があり、純粋で total な関数を表します。最下層にあるため、Tot 計算は他の Tot 計算にのみ依存でき、F* の論理コアが total のままであることを保証します。

F* のすべての項は計算型をもつように型検査されます。これは、これまで扱ってきた total な項も含みます。そのような項は既定の効果 Tot``（戻り値以外に観測可能な効果を持たない total 計算の効果）に分類されます。``e:t という型付けに与えた意味は e:Tot t にも拡張されます。例えば、e:Tot t なら、実行時に e は停止し型 t の値を生成します。さらに効果ラベルが Tot なので、副作用はありません。

実際、すでに こちら で見たように、記法上 x:t0 -> t1 は x:t0 -> Tot t1 の省略形です（t1 自体が関数型でも構いません）。より一般には、F* の関数型は x:t -> C の形を取り、引数型 t と、本体の計算型 C``（``x に依存してよい）を表します。

同様に、let 定義で見てきた戻り型注釈も省略記法です。例えば id 関数 は

let id (a:Type) (x:a) : a = x

次の省略形です。

let id (a:Type) (x:a) : Tot a = x  //the return type annotation is a computation type

したがって id の型は a:Type -> a -> Tot a です。一般に、let 定義の戻り型注釈は計算型 C になります。

明示注釈の構文 e <: t は少し挙動が異なります。F* は e <: C と書くことを許し、e が本当に計算型 C を持つか検査します。しかし効果ラベルを省いて e <: t と書いた場合は e <: _ t と解釈され、省略された効果ラベルは F* により推論され、既定で Tot にはなりません。

評価順序

純粋関数においては、評価順序は本質的ではありません。 [1] F* には純粋項を遅延評価／値呼びのいずれでも解釈できる抽象機械があり（F* の正規化器 の章を参照）、また純粋プログラムを OCaml にコンパイルする場合は、純粋項について OCaml の値呼び意味論を継承します。

効果のある引数で関数呼び出しを評価する際には、関数呼び出しの前に引数をまず値まで簡約し、その過程で（もしあれば）効果を発現させます。すなわち e1 と e2 が効果を持つ可能性があるとき、適用 e1 e2 は bind f = e1 in bind x = e2 in f x に相当します。実際、どちらかが非自明な効果を持ちうる場合、F* は内部的に e1 e2 をそのように精密化します。したがって効果のある項については、F* は左から右への 値呼び の意味論を強制します。

関数に渡されるのは計算が返す値だけなので、F* の関数の引数型は常に値型です。つまり常に t -> C の形になります。計算そのものを引数として渡したい場合は、計算を関数に包む必要があります。例えば C -> C の代わりに (unit -> C) -> C' と書きます。F* では関数（本体が非自明な効果を持つものも含め）が第一級値なので、常にこの手法が使えます。

[1]

F* は拡張的型理論であるため、純粋な F* の項は 弱正規化 にとどまります。すなわち、ある評価戦略（例えば到達不能なコードパスの深部にある再帰関数を繰り返し簡約する戦略）では停止しないことがあります。しかし閉じた純粋項については、完全に簡約できる評価戦略が必ず存在します。したがって純粋関数に関しては、特定の評価順序の選択が非停止を引き起こしうることを除けば、評価順序は本質的ではありません。
